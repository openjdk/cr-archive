1163 Enables recommended checks for problems in documentation comments.

1165 .RE
1166 .TP
1167 .B \f[CB]\-Xdoclint:\f[R](\f[CB]all\f[R]|\f[CB]none\f[R]|[\f[CB]\-\f[R]]\f[I]group\f[R])
1168 Enable or disable specific checks for bad references, accessibility
1169 issues, missing documentation comments, errors in documentation comment
1170 syntax and missing HTML tags.
1171 .RS
1172 .PP
1173 This option enables the \f[CB]javadoc\f[R] tool to check for all
1174 documentation comments included in the generated output.
1175 You can select which items to include in the generated output with the
1176 standard options \f[CB]\-public\f[R], \f[CB]\-protected\f[R],
1177 \f[CB]\-package\f[R] and \f[CB]\-private\f[R].
1178 .PP
1179 When the \f[CB]\-Xdoclint\f[R] option is enabled, it reports issues with
1180 messages similar to the \f[CB]javac\f[R] command.
1181 The \f[CB]javadoc\f[R] tool prints a message, a copy of the source line,
1182 and a caret pointing at the exact position where the error was detected.
1183 Messages may be either warnings or errors, depending on their severity
1184 and the likelihood to cause an error if the generated documentation were
1185 to be run through a validator.
1186 For example: missing documentation comments, duplicate information, and
1187 extraneous comments do not cause the \f[CB]javadoc\f[R] tool to generate
1188 invalid HTML, so these issues are reported as warnings; syntax errors,
1189 missing required HTML end tags, and references to missing or misspelled
1190 elements cause the \f[CB]javadoc\f[R] tool to generate invalid output, so
1191 these issues are reported as errors.
1192 .PP
1193 \f[CB]\-Xdoclint\f[R] option validates input comments based upon the
1194 requested markup.
1195 .PP
1196 By default, the \f[CB]\-Xdoclint\f[R] option is enabled.
1197 Disable it with the option \f[CB]\-Xdoclint:none\f[R].
1198 .PP
1199 The following options change what the \f[CB]\-Xdoclint\f[R] option
1200 reports:
1201 .IP \[bu] 2
1202 \f[CB]\-Xdoclint\ none\f[R]: Disables the \f[CB]\-Xdoclint\f[R] option
1203 .IP \[bu] 2
1204 \f[CB]\-Xdoclint\f[R] \f[I]group\f[R]: Enables \f[I]group\f[R] checks
1205 .IP \[bu] 2
1206 \f[CB]\-Xdoclint\ all\f[R]: Enables all groups of checks
1207 .IP \[bu] 2
1208 \f[CB]\-Xdoclint\ all,\-\f[R]\f[I]group\f[R]: Enables all checks except
1209 \f[I]group\f[R] checks
1210 .PP
1211 The \f[I]group\f[R] variable has one of the following values:
1212 .IP \[bu] 2
1213 \f[CB]accessibility\f[R]: Checks for the issues to be detected by an
1214 accessibility checker (for example, no caption or summary attributes
1215 specified in a \f[CB]<table>\f[R] tag).
1216 .IP \[bu] 2
1217 \f[CB]html\f[R]: Detects high\-level HTML issues, such as putting block
1218 elements inside inline elements, or not closing elements that require an
1219 end tag.
1220 The rules are derived from the \f[B]HTML 4 Specification\f[R]
1221 [https://www.w3.org/TR/html4/] or the \f[B]HTML 5 Specification\f[R]
1222 [http://www.w3.org/TR/2014/REC\-html5\-20141028/] based on the standard
1223 doclet \f[CB]html\f[R] output generation selected.
1224 This type of check enables the \f[CB]javadoc\f[R] tool to detect HTML
1225 issues that some browsers might not interpret as intended.
1226 .IP \[bu] 2
1227 \f[CB]missing\f[R]: Checks for missing documentation comments or tags (for
1228 example, a missing comment or class, or a missing \f[CB]\@return\f[R] tag
1229 or similar tag on a method).
1230 .IP \[bu] 2
1231 \f[CB]reference\f[R]: Checks for issues relating to the references to Java
1232 API elements from documentation comment tags (for example, item not
1233 found in \f[CB]\@see\f[R], or a bad name after \f[CB]\@param)\f[R].
1234 .IP \[bu] 2
1235 \f[CB]syntax\f[R]: Checks for low level issues like unescaped angle
1236 brackets (\f[CB]<\f[R] and \f[CB]>\f[R]) and ampersands (\f[CB]&\f[R]) and
1237 invalid documentation comment tags.
1238 .PP
1239 You can specify the \f[CB]\-Xdoclint\f[R] option multiple times to enable
1240 the option to check errors and warnings in multiple categories.
1241 Alternatively, you can specify multiple error and warning categories by
1242 using the preceding options.
1243 For example, use either of the following commands to check for the HTML,
1244 syntax, and accessibility issues in the file \f[I]filename\f[R].
1245 .RS
1246 .PP
1247 \f[CB]javadoc\ \-Xdoclint:html\ \-Xdoclint:syntax\ \-Xdoclint:accessibility\f[R]
1248 \f[I]filename\f[R]
1249 .RE
1250 .RS
1251 .PP
1252 \f[CB]javadoc\ \-Xdoclint:html,syntax,accessibility\f[R] \f[I]filename\f[R]
1253 .RE
