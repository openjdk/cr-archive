  27 .SH NAME
  28 .PP
  29 jshell \- interactively evaluate declarations, statements, and
  30 expressions of the Java programming language in a read\-eval\-print loop
  31 (REPL)
  32 .SH SYNOPSIS
  33 .PP
  34 \f[CB]jshell\f[R] [\f[I]options\f[R]] [\f[I]load\-files\f[R]]
  35 .TP
  36 .B \f[I]options\f[R]
  37 Command\-line options, separated by spaces.
  38 See \f[B]Options for jshell\f[R].
  39 .RS
  40 .RE
  41 .TP
  42 .B \f[I]load\-files\f[R]
  43 One or more scripts to run when the tool is started.
  44 Scripts can contain any valid code snippets or JShell commands.
  45 .RS
  46 .PP
  47 The script can be a local file or one of following predefined scripts:
  48 .TP
  49 .B \f[CB]DEFAULT\f[R]
  50 Loads the default entries, which are commonly used as imports.
  51 .RS
  52 .RE
  53 .TP
  54 .B \f[CB]JAVASE\f[R]
  55 Imports all Java SE packages.
  56 .RS
  57 .RE
  58 .TP
  59 .B \f[CB]PRINTING\f[R]
  60 Defines \f[CB]print\f[R], \f[CB]println\f[R], and \f[CB]printf\f[R] as
  61 \f[CB]jshell\f[R] methods for use within the tool.
  62 .RS
  63 .RE
  64 .PP
  65 For more than one script, use a space to separate the names.
  66 Scripts are run in the order in which they\[aq]re entered on the command
  67 line.
  68 Command\-line scripts are run after startup scripts.
  69 To run a script after JShell is started, use the \f[CB]/open\f[R] command.
  70 .PP
  71 To accept input from standard input and suppress the interactive I/O,
  72 enter a hyphen (\-) for \f[I]load\-files\f[R].
  73 This option enables the use of the \f[CB]jshell\f[R] tool in pipe chains.
  74 .RE
  75 .SH DESCRIPTION
  76 .PP
  77 JShell provides a way to interactively evaluate declarations,
  78 statements, and expressions of the Java programming language, making it
  79 easier to learn the language, explore unfamiliar code and APIs, and
  80 prototype complex code.
  81 Java statements, variable definitions, method definitions, class
  82 definitions, import statements, and expressions are accepted.
  83 The bits of code entered are called snippets.
  84 .PP
  85 As snippets are entered, they\[aq]re evaluated, and feedback is
  86 provided.
  87 Feedback varies from the results and explanations of actions to nothing,
  88 depending on the snippet entered and the feedback mode chosen.
  89 Errors are described regardless of the feedback mode.
  90 Start with the verbose mode to get the most feedback while learning the
  91 tool.
  92 .PP
  93 Command\-line options are available for configuring the initial
  94 environment when JShell is started.
  95 Within JShell, commands are available for modifying the environment as
  96 needed.
  97 .PP
  98 Existing snippets can be loaded from a file to initialize a JShell
  99 session, or at any time within a session.
 100 Snippets can be modified within the session to try out different
 101 variations and make corrections.
 102 To keep snippets for later use, save them to a file.

1100 .SS Command abbreviations
1101 .PP
1102 An abbreviation of a command is accepted if the abbreviation uniquely
1103 identifies a command.
1104 For example, \f[CB]/l\f[R] is recognized as the \f[CB]/list\f[R] command.
1105 However, \f[CB]/s\f[R] isn\[aq]t a valid abbreviation because it can\[aq]t
1106 be determined if the \f[CB]/set\f[R] or \f[CB]/save\f[R] command is meant.
1107 Use \f[CB]/se\f[R] for the \f[CB]/set\f[R] command or \f[CB]/sa\f[R] for the
1108 \f[CB]/save\f[R] command.
1109 .PP
1110 Abbreviations are also accepted for subcommands, command arguments, and
1111 command options.
1112 For example, use \f[CB]/m\ \-a\f[R] to display all methods.
1113 .SS History navigation
1114 .PP
1115 A history of what was entered is maintained across sessions.
1116 Use the up and down arrows to scroll through commands and snippets from
1117 the current and past sessions.
1118 Use the Ctrl key with the up and down arrows to skip all but the first
1119 line of multiline snippets.
1120 .SS History search
1121 .PP
1122 Use the Ctrl+R key combination to search the history for the string
1123 entered.
1124 The prompt changes to show the string and the match.
1125 Ctrl+R searches backwards from the current location in the history
1126 through earlier entries.
1127 Ctrl+S searches forward from the current location in the history though
1128 later entries.

1134 The Ctrl key and Meta key are used in key combinations.
1135 If your keyboard doesn\[aq]t have a Meta key, then the Alt key is often
1136 mapped to provide Meta key functionality.

1218 .SH EXAMPLE OF STARTING AND STOPPING A JSHELL SESSION
1219 .PP
1220 JShell is provided with the JDK.
1221 To start a session, enter \f[CB]jshell\f[R] on the command line.
1222 A welcome message is printed, and a prompt for entering commands and
1223 snippets is provided.
1224 .IP
1225 .nf
1226 \f[CB]
1227 %\ jshell
1228 |\ \ Welcome\ to\ JShell\ \-\-\ Version\ 9
1229 |\ \ For\ an\ introduction\ type:\ /help\ intro
1230 
1231 jshell>
1232 \f[R]
1233 .fi
1234 .PP
1235 To see which snippets were automatically loaded when JShell started, use
1236 the \f[CB]/list\ \-start\f[R] command.
1237 The default startup snippets are import statements for common packages.
1238 The ID for each snippet begins with the letter \f[I]s\f[R], which
1239 indicates it\[aq]s a startup snippet.
1240 .IP
1241 .nf
1242 \f[CB]
1243 jshell>\ /list\ \-start
1244 
1245 \ \ s1\ :\ import\ java.io.*;
1246 \ \ s2\ :\ import\ java.math.*;
1247 \ \ s3\ :\ import\ java.net.*;
1248 \ \ s4\ :\ import\ java.nio.file.*;
1249 \ \ s5\ :\ import\ java.util.*;
1250 \ \ s6\ :\ import\ java.util.concurrent.*;
1251 \ \ s7\ :\ import\ java.util.function.*;
1252 \ \ s8\ :\ import\ java.util.prefs.*;
1253 \ \ s9\ :\ import\ java.util.regex.*;
1254 \ s10\ :\ import\ java.util.stream.*;
1255 
1256 jshell>
1257 \f[R]
1258 .fi
1259 .PP
1260 To end the session, use the \f[CB]/exit\f[R] command.
1261 .IP
1262 .nf
1263 \f[CB]
1264 jshell>\ /exit
1265 |\ \ Goodbye
1266 
1267 %
1268 \f[R]
1269 .fi
1270 .SH EXAMPLE OF ENTERING SNIPPETS
1271 .PP
1272 Snippets are Java statements, variable definitions, method definitions,
1273 class definitions, import statements, and expressions.
1274 Terminating semicolons are automatically added to the end of a completed
1275 snippet if they\[aq]re missing.
1276 .PP
1277 The following example shows two variables and a method being defined,
1278 and the method being run.
1279 Note that a scratch variable is automatically created to hold the result
1280 because no variable was provided.
1281 .IP
1282 .nf
1283 \f[CB]
1284 jshell>\ int\ a=4
1285 a\ ==>\ 4
1286 
1287 jshell>\ int\ b=8
1288 b\ ==>\ 8
1289 
1290 jshell>\ int\ square(int\ i1)\ {
1291 \ \ \ ...>\ return\ i1\ *\ i1;
1292 \ \ \ ...>\ }
1293 |\ \ created\ method\ square(int)
1294 
1295 jshell>\ square(b)
1296 $5\ ==>\ 64
1297 \f[R]
1298 .fi
1299 .SH EXAMPLE OF CHANGING SNIPPETS
1300 .PP
1301 Change the definition of a variable, method, or class by entering it
1302 again.
1303 .PP
1304 The following examples shows a method being defined and the method run:
1305 .IP
1306 .nf
1307 \f[CB]
1308 jshell>\ String\ grade(int\ testScore)\ {
1309 \ \ \ ...>\ \ \ \ \ if\ (testScore\ >=\ 90)\ {
1310 \ \ \ ...>\ \ \ \ \ \ \ \ \ return\ "Pass";
1311 \ \ \ ...>\ \ \ \ \ }
1312 \ \ \ ...>\ \ \ \ \ return\ "Fail";
1313 \ \ \ ...>\ }
1314 |\ \ created\ method\ grade(int)
1315 
1316 jshell>\ grade(88)
1317 $3\ ==>\ "Fail"
1318 \f[R]
1319 .fi
1320 .PP
1321 To change the method \f[CB]grade\f[R] to allow more students to pass,
1322 enter the method definition again and change the pass score to
1323 \f[CB]80\f[R].
1324 Use the up arrow key to retrieve the previous entries to avoid having to
1325 reenter them and make the change in the \f[CB]if\f[R] statement.
1326 The following example shows the new definition and reruns the method to
1327 show the new result:
1328 .IP
1329 .nf
1330 \f[CB]
1331 jshell>\ String\ grade(int\ testScore)\ {
1332 \ \ \ ...>\ \ \ \ \ if\ (testScore\ >=\ 80)\ {
1333 \ \ \ ...>\ \ \ \ \ \ \ \ \ return\ "Pass";
1334 \ \ \ ...>\ \ \ \ \ }
1335 \ \ \ ...>\ \ \ \ \ return\ "Fail";
1336 \ \ \ ...>\ }
1337 |\ \ modified\ method\ grade(int)
1338 
1339 jshell>\ grade(88)
1340 $5\ ==>\ "Pass"
1341 \f[R]
1342 .fi
1343 .PP
1344 For snippets that are more than a few lines long, or to make more than a
1345 few changes, use the \f[CB]/edit\f[R] command to open the snippet in an
1346 editor.
1347 After the changes are complete, close the edit window to return control
1348 to the JShell session.
1349 The following example shows the command and the feedback provided when
1350 the edit window is closed.
1351 The \f[CB]/list\f[R] command is used to show that the pass score was
1352 changed to \f[CB]85\f[R].
1353 .IP
1354 .nf
1355 \f[CB]
1356 jshell>\ /edit\ grade
1357 |\ \ modified\ method\ grade(int)
1358 jshell>\ /list\ grade
1359 
1360 \ \ \ 6\ :\ String\ grade(int\ testScore)\ {
1361 \ \ \ \ \ \ \ \ \ \ \ if\ (testScore\ >=\ 85)\ {
1362 \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ return\ "Pass";
1363 \ \ \ \ \ \ \ \ \ \ \ }
1364 \ \ \ \ \ \ \ \ \ \ \ return\ "Fail";
1365 \ \ \ \ \ \ \ }
1366 \f[R]
1367 .fi
1368 .SH EXAMPLE OF CREATING A CUSTOM FEEDBACK MODE
1369 .PP
1370 The feedback mode determines the prompt that\[aq]s displayed, the
1371 feedback messages that are provided as snippets are entered, and the
1372 maximum length of a displayed value.
1373 Predefined feedback modes are provided.
1374 Commands for creating custom feedback modes are also provided.
1375 .PP
1376 Use the \f[CB]/set\ mode\f[R] command to create a new feedback mode.
1377 In the following example, the new mode \f[CB]mymode\f[R], is based on the
1378 predefined feedback mode, \f[CB]normal\f[R], and verifying command
1379 feedback is displayed:
1380 .IP
1381 .nf
1382 \f[CB]
1383 jshell>\ /set\ mode\ mymode\ normal\ \-command
1384 |\ \ Created\ new\ feedback\ mode:\ mymode
1385 \f[R]
1386 .fi
1387 .PP
1388 Because the new mode is based on the \f[CB]normal\f[R] mode, the prompts
1389 are the same.
1390 The following example shows how to see what prompts are used and then
1391 changes the prompts to custom strings.
1392 The first string represents the standard JShell prompt.
1393 The second string represents the prompt for additional lines in
1394 multiline snippets.
1395 .IP
1396 .nf
1397 \f[CB]
1398 jshell>\ /set\ prompt\ mymode
1399 |\ \ /set\ prompt\ mymode\ "\\njshell>\ "\ "\ \ \ ...>\ "
1400 
1401 jshell>\ /set\ prompt\ mymode\ "\\nprompt$\ "\ "\ \ \ continue$\ "
1402 \f[R]
1403 .fi
1404 .PP
1405 The maximum length of a displayed value is controlled by the truncation
1406 setting.
1407 Different types of values can have different lengths.
1408 The following example sets an overall truncation value of 72, and a
1409 truncation value of 500 for variable value expressions:
1410 .IP
1411 .nf
1412 \f[CB]
1413 jshell>\ /set\ truncation\ mymode\ 72
1414 
1415 jshell>\ /set\ truncation\ mymode\ 500\ varvalue
1416 \f[R]
1417 .fi
1418 .PP
1419 The feedback displayed after snippets are entered is controlled by the
1420 format setting and is based on the type of snippet entered and the
1421 action taken for that snippet.
1422 In the predefined mode \f[CB]normal\f[R], the string \f[CB]created\f[R] is
1423 displayed when a method is created.
1424 The following example shows how to change that string to
1425 \f[CB]defined\f[R]:
1426 .IP
1427 .nf
1428 \f[CB]
1429 jshell>\ /set\ format\ mymode\ action\ "defined"\ added\-primary
1430 \f[R]
1431 .fi
1432 .PP
1433 Use the \f[CB]/set\ feedback\f[R] command to start using the feedback mode
1434 that was just created.
1435 The following example shows the custom mode in use:
1436 .IP
1437 .nf
1438 \f[CB]
1439 jshell>\ /set\ feedback\ mymode
1440 |\ \ Feedback\ mode:\ mymode
1441 
1442 prompt$\ int\ square\ (int\ num1){
1443 \ \ \ continue$\ return\ num1*num1;
1444 \ \ \ continue$\ }
1445 |\ \ defined\ method\ square(int)
1446 
1447 prompt$
1448 \f[R]
1449 .fi
