 28 jps \- list the instrumented JVMs on the target system

 77 .SH DESCRIPTION
 78 .PP
 79 The \f[CB]jps\f[R] command lists the instrumented Java HotSpot VMs on the
 80 target system.
 81 The command is limited to reporting information on JVMs for which it has
 82 the access permissions.
 83 .PP











 84 If the \f[CB]jps\f[R] command is run without specifying a \f[CB]hostid\f[R],
 85 then it searches for instrumented JVMs on the local host.
 86 If started with a \f[CB]hostid\f[R], then it searches for JVMs on the
 87 indicated host, using the specified protocol and port.
 88 A \f[CB]jstatd\f[R] process is assumed to be running on the target host.
 89 .PP
 90 The \f[CB]jps\f[R] command reports the local JVM identifier, or
 91 \f[CB]lvmid\f[R], for each instrumented JVM found on the target system.
 92 The \f[CB]lvmid\f[R] is typically, but not necessarily, the operating
 93 system\[aq]s process identifier for the JVM process.
 94 With no options, the \f[CB]jps\f[R] command lists each Java
 95 application\[aq]s \f[CB]lvmid\f[R] followed by the short form of the
 96 application\[aq]s class name or jar file name.
 97 The short form of the class name or JAR file name omits the class\[aq]s
 98 package information or the JAR files path information.
 99 .PP
100 The \f[CB]jps\f[R] command uses the Java launcher to find the class name
101 and arguments passed to the main method.
102 If the target JVM is started with a custom launcher, then the class or
103 JAR file name, and the arguments to the \f[CB]main\f[R] method aren\[aq]t
104 available.
105 In this case, the \f[CB]jps\f[R] command outputs the string
106 \f[CB]Unknown\f[R] for the class name, or JAR file name, and for the
107 arguments to the \f[CB]main\f[R] method.
108 .PP
109 The list of JVMs produced by the \f[CB]jps\f[R] command can be limited by
110 the permissions granted to the principal running the command.
111 The command lists only the JVMs for which the principal has access
112 rights as determined by operating system\-specific access control
113 mechanisms.
114 .SH HOST IDENTIFIER
115 .PP
116 The host identifier, or \f[CB]hostid\f[R], is a string that indicates the
117 target system.
118 The syntax of the \f[CB]hostid\f[R] string corresponds to the syntax of a
119 URI:
120 .RS
121 .PP
122 [\f[I]protocol\f[R]\f[CB]:\f[R]][[\f[CB]//\f[R]]\f[I]hostname\f[R]][\f[CB]:\f[R]\f[I]port\f[R]][\f[CB]/\f[R]\f[I]servername\f[R]]
123 .RE
124 .TP
125 .B \f[I]protocol\f[R]
126 The communications protocol.
127 If the \f[I]protocol\f[R] is omitted and a \f[I]hostname\f[R] isn\[aq]t
128 specified, then the default protocol is a platform\-specific, optimized,
129 local protocol.
130 If the protocol is omitted and a host name is specified, then the
131 default protocol is \f[CB]rmi\f[R].
132 .RS
133 .RE
134 .TP
135 .B \f[I]hostname\f[R]
136 A host name or IP address that indicates the target host.
137 If you omit the \f[I]hostname\f[R] parameter, then the target host is the
138 local host.
139 .RS
140 .RE
141 .TP
142 .B \f[I]port\f[R]
143 The default port for communicating with the remote server.
144 If the \f[I]hostname\f[R] parameter is omitted or the \f[I]protocol\f[R]
145 parameter specifies an optimized, local protocol, then the \f[I]port\f[R]
146 parameter is ignored.
147 Otherwise, treatment of the \f[I]port\f[R] parameter is
148 implementation\-specific.
149 For the default \f[CB]rmi\f[R] protocol, the \f[I]port\f[R] parameter
150 indicates the port number for the \f[CB]rmiregistry\f[R] on the remote
151 host.
152 If the \f[I]port\f[R] parameter is omitted, and the \f[I]protocol\f[R]
153 parameter indicates \f[CB]rmi\f[R], then the default \f[CB]rmiregistry\f[R]
154 port (\f[CB]1099\f[R]) is used.
155 .RS
156 .RE
157 .TP
158 .B \f[I]servername\f[R]
159 The treatment of this parameter depends on the implementation.
160 For the optimized, local protocol, this field is ignored.
161 For the \f[CB]rmi\f[R] protocol, this parameter is a string that
162 represents the name of the RMI remote object on the remote host.
163 See the \f[B]jstatd\f[R] command \f[CB]\-n\f[R] option.
164 .RS
165 .RE
166 .SH OUTPUT FORMAT OF THE JPS COMMAND
167 .PP
168 The output of the \f[CB]jps\f[R] command has the following pattern:
169 .RS
170 .PP
171 \f[I]lvmid\f[R] [ [ \f[I]classname\f[R] | \f[I]JARfilename\f[R] |
172 \f[CB]"Unknown"\f[R]] [ \f[I]arg\f[R]* ] [ \f[I]jvmarg\f[R]* ] ]
173 .RE
174 .PP
175 All output tokens are separated by white space.
176 An \f[CB]arg\f[R] value that includes embedded white space introduces
177 ambiguity when attempting to map arguments to their actual positional
178 parameters.
179 .PP
180 \f[B]Note:\f[R]
181 .PP
182 It\[aq]s recommended that you don\[aq]t write scripts to parse
183 \f[CB]jps\f[R] output because the format might change in future releases.
184 If you write scripts that parse \f[CB]jps\f[R] output, then expect to
185 modify them for future releases of this tool.
186 .SH EXAMPLES
187 .PP
188 This section provides examples of the \f[CB]jps\f[R] command.
189 .PP
190 List the instrumented JVMs on the local host:
191 .IP
192 .nf
193 \f[CB]
194 jps
195 18027\ Java2Demo.JAR
196 18032\ jps
197 18005\ jstat
198 \f[R]
199 .fi
200 .PP
201 The following example lists the instrumented JVMs on a remote host.
202 This example assumes that the \f[CB]jstat\f[R] server and either the its
203 internal RMI registry or a separate external \f[CB]rmiregistry\f[R]
204 process are running on the remote host on the default port (port
205 \f[CB]1099\f[R]).
206 It also assumes that the local host has appropriate permissions to
207 access the remote host.
208 This example includes the \f[CB]\-l\f[R] option to output the long form of
209 the class names or JAR file names.
210 .IP
211 .nf
212 \f[CB]
213 jps\ \-l\ remote.domain
214 3002\ /opt/jdk1.7.0/demo/jfc/Java2D/Java2Demo.JAR
215 2857\ sun.tools.jstatd.jstatd
216 \f[R]
217 .fi
218 .PP
219 The following example lists the instrumented JVMs on a remote host with
220 a nondefault port for the RMI registry.
221 This example assumes that the \f[CB]jstatd\f[R] server, with an internal
222 RMI registry bound to port \f[CB]2002\f[R], is running on the remote host.
223 This example also uses the \f[CB]\-m\f[R] option to include the arguments
224 passed to the \f[CB]main\f[R] method of each of the listed Java
225 applications.
226 .IP
227 .nf
228 \f[CB]
229 jps\ \-m\ remote.domain:2002
230 3002\ /opt/jdk1.7.0/demo/jfc/Java2D/Java2Demo.JAR
231 3102\ sun.tools.jstatd.jstatd\ \-p\ 2002
232 \f[R]
233 .fi
