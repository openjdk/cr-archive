 26 .SH NAME
 27 .PP
 28 jinfo \- generate Java configuration information for a specified Java
 29 process
 30 .SH SYNOPSIS
 31 .PP
 32 \f[B]Note:\f[R] This command is experimental and unsupported.
 33 .PP
 34 \f[CB]jinfo\f[R] [\f[I]option\f[R]] \f[I]pid\f[R]
 35 .TP
 36 .B \f[I]option\f[R]
 37 This represents the \f[CB]jinfo\f[R] command\-line options.
 38 See \f[B]Options for the jinfo Command\f[R].
 39 .RS
 40 .RE
 41 .TP
 42 .B \f[I]pid\f[R]
 43 The process ID for which the configuration information is to be printed.
 44 The process must be a Java process.
 45 To get a list of Java processes running on a machine, use either the
 46 \f[CB]ps\f[R] command or, if the JVM processes are not running in a
 47 separate docker instance, the \f[B]jps\f[R] command.
 48 .RS








 49 .RE
 50 .SH DESCRIPTION
 51 .PP
 52 The \f[CB]jinfo\f[R] command prints Java configuration information for a
 53 specified Java process.
 54 The configuration information includes Java system properties and JVM
 55 command\-line flags.
 56 If the specified process is running on a 64\-bit JVM, then you might
 57 need to specify the \f[CB]\-J\-d64\f[R] option, for example:
 58 .RS
 59 .PP
 60 \f[CB]jinfo\ \-J\-d64\ \-sysprops\f[R] \f[I]pid\f[R]
 61 .RE
 62 .PP
 63 This command is unsupported and might not be available in future
 64 releases of the JDK.
 65 In Windows Systems where \f[CB]dbgeng.dll\f[R] is not present, the
 66 Debugging Tools for Windows must be installed to have these tools work.
 67 The \f[CB]PATH\f[R] environment variable should contain the location of
 68 the \f[CB]jvm.dll\f[R] that\[aq]s used by the target process or the
 69 location from which the core dump file was produced.
 70 .SH OPTIONS FOR THE JINFO COMMAND
 71 .PP
 72 \f[B]Note:\f[R]
 73 .PP
 74 If none of the following options are used, both the command\-line flags
 75 and the system property name\-value pairs are printed.
 76 .TP
 77 .B \f[CB]\-flag\f[R] \f[I]name\f[R]
 78 Prints the name and value of the specified command\-line flag.

 80 .RE

 90 .RE
