 23 
 24 This directory contains source files of NSK tests framework
 25 shared between all NSK tests.
 26 
 27 Files located directly in this directory provide general support
 28 for all tests.
 29 
 30 Files in the subdirectories provide specific support for tests of
 31 particular subsuites.
 32 
 33 ---------------------------------------------------------------------------------
 34 
 35 Short description of files:
 36 
 37     common exceptions:
 38         Failure,java, TestBug.java, Oddity.java
 39     common constants:
 40         Consts.java
 41     parsing command line arguments:
 42         ArgumentPareser.java
 43     output of errors and messages:
 44         Log.java
 45     process running:
 46         LocalProcess.java, IORedirector.java
 47     class loading/unloading:
 48         DummyClassLoader.java, ZipClassLoader.java,
 49         CustomClassLoader.java, ClassUnloder.java
 50     objects finalization:
 51         Finalizable.java, FinalizableObject.java, Finalizer.java
 52     threads synchronization:
 53         Wicket.java
 54     text processing:
 55         Grep.java, Paragrep.java
 56     timeouts handling:
 57         Harakiri.java, TimeoutHandler.java
 58     tree structures support:
 59         Denotation.java, TreeNodesDenotation.java
 60     RAS mode support:
 61         RASagent.java, JVMTIagent.c
 62     JVMDI tests support:
 63         JVMDITools.h, JVMDITools.c
 64 
 65 Short description of subdirectories:
 66 
 67     Alien       - support for accessing external tests (JCK)
 68     native      - support for native part of NSK tests
 69     jni         - support for JNI tests and accessing JNI API
 70     jvmti       - support for JVMTI tests and accessing JVMTI API
 71     jpda        - support for two-VMs JPDA tests
 72     jdwp        - support for JDWP tests and accessing JDWP API
 73     jdi         - support for JDI tests and accesing JDI API
 74     jdb         - support for JDB tests and accessing JDB tool
 75     monitoring  - support for monitoring tests and accessing Java Monitoring&Management API
 76     sysdict     - support for System Dictionary tests
 77     gc          - support for GC tests
 78     regression  - support for regression tests for known bugs
 79     split_verifier - support for Split Verifier tests
 80 
 81 For more detailed description see README files in subdirectories.

 85 Naming conventions
 86 
 87 Classes:
 88 
 89     All shared classes are groupped into packages to prevent
 90     name collision.
 91 
 92     All classes exported directly from this directory are
 93     of package:
 94 
 95         nsk.share
 96 
 97     All classes exported from subdirectories are of particular
 98     subpackage, e.g.:
 99 
100         nsk.share.jpda
101         nsk.share.jdwp
102         nsk.share.jdi
103         nsk.share.jdb
104         nsk.share.sysdict
105 
106 Native functions and macroses:
107 
108     Most native functions have special prefix to prevent linking collisions.
109     Most macroses also have special prefix and are wrote in upper register.
110 
111     Here is typical naming scheme used for native functions and macroses:
112 
113         share/native
114             functions: nsk_*
115             macroses:  NSK_*
116 
117         share/jni
118             functions: nsk_jni_*
119             macroses:  NSK_JNI_*
120 
121         share/jvmti
122             functions: nsk_jvmti_*
123             macroses:  NSK_JVMTI_*
124 
125     However, some native functions and macroses do not follow this scheme,
126     in order to preserve compatibility with old tests.
